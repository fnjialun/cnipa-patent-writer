# 专利配图生成（白底·纯黑·大字·可读）

专利附图惯例：**白底、纯黑实线、黑字、无颜色无灰阶**。核心目标是**可读**——嵌入文档缩放后每个字、每条线都清楚。

> **图随内容定，不照搬模板的图。** 画几张、画哪些类型，**由本技术方案本身决定**——不是每篇都要 5 张图，也不一定
> 有架构图或 JSON 图。先想清楚"这篇发明需要哪些图把方法/结构/数据讲清楚"，再画对应的图；模板的图只参考**画风
> （纯黑白）和排版**，画的是什么结构不要搬，**也别固定"图1=流程、图5=架构、图3图4=JSON"这种编排**。

## 〇、六条硬规则（每张图都要满足——这是返工最多的地方）

1. **纯黑、线要粗**：所有线/箭头/文字 `color='black'`，线宽 **lw≥1.8**；保存 `facecolor='white'`。细灰线（lw<1.5）
   缩放后发灰、像"没黑透"——这正是"配图不纯黑"的根因。别用任何彩色/灰度填充。
2. **框随文字自适应、文字绝不溢出**：盒子大小要**由文字撑出来**（按显示宽度换行到框内、框高随行数长），
   **不要先定死框尺寸再硬塞文字**（那必溢出）。文字四周留白，不贴边、不超出框线。
3. **竖向单列排版**：流程/模块一律**自上而下单列**——满版宽的框、字才大；**不要左右并排成网格**（一行挤两三个框
   →每个框窄→字必小）。单列还有个好处：箭头只在框之间直走，**线永远不穿过框**。
4. **文字不压线**：任何文字（箭头标注、分组标签、曲线注释）**要么带白底、要么离线有明显间距**，绝不压在线条上。
   **不要用竖排单字标签**（如把"环境回采"竖着排）——难读又易压线；要标就横排、放在留白处。
5. **线条/文字都完整显示**：不能有线被遮/被吞、不能有文字被框或图边切掉、不能有文字超出其所属线条/框的范围。
   **靠"白底盒子盖住穿过的线"是下策**（用户明确不接受这种遮盖）；正解是**走线干净**——单列布局让线本就不必穿框。
6. **图文匹配、字够大**：画布宽贴近页宽（**~6.4in**），嵌入（~15.3cm）后近 1:1，正文字号 ~13pt → 嵌入后 ~12pt，
   与正文齐平。**别画又大又空、字却很小的图**（画布太宽 / 框太大字太小都属此病）。高度随内容自适应（可占大半页）。

## 一、强制自查（不可跳过——可靠出图全靠这步）

**每画完一张图，必须单独 `Read` 这张 PNG，逐条对照上面六条核对，不合格就改了重画，再嵌入。**
这一步是"图稳定好看"与"碰运气"的分水岭：同一套工具，认真逐张看图并迭代的就出好图，省掉这步的就出溢出/压线/小字。
对照清单（缺一不可）：
- [ ] 纯黑、线够粗，无彩色/发灰；
- [ ] 每个框的文字都在框内、四周有留白，无溢出、无贴边；
- [ ] 单列竖排，无左右并排导致的小字；线不穿框；
- [ ] 无文字压线；无竖排单字标签；
- [ ] 所有线条完整、无被吞；所有文字完整、无被切/超范围；
- [ ] 字够大（嵌入后与正文相当），图不空旷。
嵌入整篇后，第 7 步整篇渲染里**再复看一次**缩放后的真实效果。

## 二、首选画法：`make_figures.py` 的高层函数（自适应、单列、已内置六规则）

流程图、模块架构图**优先直接调用**这两个函数（别手搓固定尺寸的盒子——手搓最容易违反规则 2/3/6）：

```python
import make_figures as mf
mf.setup_cjk()
# 竖向流程：每步可为 str 或 (标题, 详述)；框高自适应、单列、字大
mf.vflow("/tmp/figN.png",
         ["接入与采集", ("特征处理", "对采集数据做……，输出特征向量"), "判定与输出"],
         down_labels={1: "满足条件"},                 # 在第 1→2 箭头旁标注(带白底);别用右侧菱形旁注
         title="图N 整体流程示意图")
# 竖向模块架构：可选 groups 套虚线分组、feedback 画干净回流箭头(标签横排白底、不压线)
mf.vmodules("/tmp/figN.png",
            [("A模块", "职责……"), ("B模块", "职责……"), ("C模块", "职责……")],
            groups=[("某子系统", 0, 1)], feedback=(2, 0, "结果回流"),
            title="图N 系统模块框图")
```
这两个函数已保证：框随文字自适应不溢出、单列竖排、lw=1.8 纯黑、画布贴近页宽字够大、回流/分组标签横排带白底不压线。
图号用占位「图N」，实际图号由你的附图编排定。

## 三、曲线/坐标类图（Nyquist、增量容量、控制特性…）要自己用 matplotlib 画时

这类没有现成高层函数，自己画，但**同样守六规则**，重点：
- 线 `lw≥1.8`、`color='black'`；坐标轴 lw≥1.2。
- **标注不要压在曲线上**：用带箭头的引线（`annotate(..., arrowprops=dict(arrowstyle='-|>', color='black'))`）把文字
  引到曲线旁的留白处，文字加白底 `bbox=dict(boxstyle='square', fc='white', ec='none')`。坐标轴标题可沿轴排（这是
  唯一可接受的"竖排"，因为是整行词、非单字堆叠）。
- 图要表达的**语义要对**（例：带死区的比例控制，死区内控制量应压平为 0，别把直线径直画穿死区）。
- 两子图（如谱+曲线）**上下堆叠** `subplots(2,1)`，别左右并排（会小字）。

## 四、CJK 字体（不做就豆腐块）
```python
import matplotlib, matplotlib.font_manager as fm, matplotlib.pyplot as plt
matplotlib.use("Agg")
FP = "/path/to/NotoSansSC.ttf"   # 或系统中文字体；用 fc-list :lang=zh file 找
fm.fontManager.addfont(FP); FONT = fm.FontProperties(fname=FP).get_name()
plt.rcParams["font.sans-serif"] = [FONT]; plt.rcParams["axes.unicode_minus"] = False
```
`make_figures.py` 的 `setup_cjk()` 会自动找中文字体。**含中文的代码/JSON 别用 `family="monospace"`**——DejaVu Sans
Mono 无中文字形→中文全 □；直接用 CJK 字体渲染（`fontfamily=FONT`），牺牲一点等宽换中文可显。

## 五、可读性公式（理解规则 6 的由来）
> 图内有效字号 ≈ 图内字号(pt) × (嵌入宽 in / 画布宽 in)

画布宽贴近嵌入宽（~6.4in，缩放比≈1）→ 13pt 字嵌入后仍≈12pt，清晰。反例：画布 12in、嵌入 6in → 缩放 0.5，
13pt 变 ~6.5pt，糊。这就是规则 6"图大字小"的成因——画布别画太宽。

## 六、graphviz 备选（`gen_figures_graphviz.py`）
节点多、连线复杂、想自动布局时可用 graphviz（需系统 `dot`）。`new_graph()` 已设纯黑白、`fontsize≈11`。同样守六规则：
图别太宽（否则嵌入后字小）、标签别压线、用 ASCII 别用 Unicode 下标/希腊字母（中文字体常缺字→□）。

## 七、三个配图脚本一览
- `make_figures.py`：**首选**。`vflow`/`vmodules`（自适应单列、已内置六规则）、低层图元 `box/diamond/arrow`、
  `json_examples`、`setup_cjk`。
- `gen_figures_graphviz.py`：graphviz 纯黑白流程/架构图（自动布局），含占位示例。
- `gen_json_figs.py`：结构化输出面板图（长行自动换行不溢出）；**可选**，仅当发明确有结构化输出值得展示时才用。
> 三个脚本的示例标签都是**通用占位符**，演示画法而非内容——务必替换成你这篇专利的真实步骤/模块/字段。
